Starting Point

The Prompt

Build a sandbox web application for Canadian mortgage income document ingestion and review. Documents are uploaded, text-extracted via OCR, run through a configurable AI agent pipeline (classification → extraction → validation), and presented for human review with field-level confidence scores. Every agent decision must be logged immutably for full auditability.

Requirements: 5 document types, deterministic test mode, append-only audit trail,
configurable validation rules, and confidence-based review flagging.

Capabilities

Key Features

AI Classification

Claude Sonnet 4.5 automatically identifies document type from extracted text — T4, NOA, Paystub, Bank Statement, or Employment Letter.

Field Extraction

Structured data pulled from unstructured documents with per-field confidence scores. Supports 40+ field types across 5 document categories.

Rule-Based Validation

Configurable rules catch impossible values: tax > income, negative salaries, missing required fields. YAML-overridable.

Confidence Scoring

3-factor weighted score (model 50%, validation 30%, cross-field 20%). Documents below 85% auto-flagged for human review.

Human Review Queue

Flagged documents presented with field-level confidence badges, override capability, and full audit visibility.

Immutable Audit Trail

Every agent execution logged with SHA-256 input/output hashes. Append-only — nothing deleted, full chain of custody.

Engineering Quality

What Makes It Robust

52 Automated Tests

  • Unit tests — Classification, extraction, validation, confidence engine
  • Integration tests — Full pipeline end-to-end with deterministic fixtures
  • Golden tests — Regression suite with expected outputs for T4, NOA, Employment Letter
  • Adversarial tests — Garbled text, missing fields, impossible values, multi-doc files

Deterministic Mode

Every agent has a fixture path (DETERMINISTIC_MODE=true) that returns predictable outputs without calling Claude. All 52 tests run in <400ms with zero API costs.

Append-Only Audit Trail

  • SHA-256 hash of every agent input
  • SHA-256 hash of every agent output
  • Pipeline version tracked per execution
  • Human review overrides logged separately
  • Nothing is ever deleted or modified

Configurable Rules

Validation rules defined in code with YAML override support. Add new document types or change thresholds without touching agent code. Rules enforce required fields, cross-field comparisons, and range checks.

Decision Framework

Confidence Scoring

score = (modelConfidence × 0.5) + (validationScore × 0.3) + (crossFieldScore × 0.2)

Model Confidence 50%

AI model's self-assessed accuracy for classification and extraction

Validation Score 30%

1.0 if all rules pass, 0.5 if any error-severity rule fails

Cross-Field Score 20%

1.0 minus 0.20 per error, 0.05 per warning from rule engine

Score ≥ 85% → Auto-approved   |   Score < 85% → Flagged for human review
